Security: Constitutional

The March overhaul fixed individual bugs. April made security architectural. Every research tank now runs in a hardened container: non-root (UID 1000), read-only filesystem, all capabilities dropped, custom seccomp BPF profile blocking dangerous syscalls, and no-new-privileges flag set. Tanks cannot reach the internet — they sit on an isolated Docker network with no route out. The inference proxy is the only bridge between the isolated tanks and cloud APIs, and it holds all the API keys. Tanks never see a key.

We ran a formal NIST Cybersecurity Framework v2.0 audit across all six functions: Govern, Identify, Protect, Detect, Respond, Recover. Every function passed. The full report is published on the roadmap.

Then we built RedAmon — a Rust binary that continuously red-teams the infrastructure. Every 10 minutes it probes random tanks for network isolation, checks container hardening, scans for API key leaks, and verifies the inference proxy boundary. First scan: zero findings. Security isn't a feature we added. It's the foundation everything else sits on.

The Inference Proxy: A Security Boundary

The March post described a Groq/Cerebras chain. April formalized it into a proper security boundary. The inference proxy is a multi-threaded HTTP server that sits between the isolated tank network and the internet. It rotates across 7 Cerebras keys and 8 Groq keys with per-key rate limiting and file-based locking. If all cloud keys are exhausted, it falls back to local Ollama with a non-blocking lock. Tanks call the proxy at a fixed internal IP. They never know which provider answered.

This architecture means we can swap providers, add keys, or change models without touching a single tank. The proxy is the chokepoint, and chokepoints are where you put your security.

Navigator: A Specimen Sees the Internet

Every other specimen in the Digiquarium explores offline Wikipedia through Kiwix. Navigator is different. It searches the live internet through the TinyFish API — a headless browser automation service that goes to Google, reads real articles, and returns extracted text.

Navigator started with "science discoveries breakthroughs" and followed its own curiosity through photosynthesis, plant microbiomes, sustainable agriculture, microbial carbon capture, artificial intelligence, space exploration, the Artemis II mission, Mars life support, space debris mitigation, post-quantum cryptography, and quantum metrology in robotics. Nobody told it to go there. It followed links in its own mind.

The research question: does access to live, current information produce a fundamentally different kind of personality than static encyclopedia exploration? We'll measure this through drift analysis against the Wikipedia-only specimens.

Rust on the NUC: Three Services from Source

The March system ran on 21 Python daemons. April added three compiled Rust services:

OpenFang v0.1.0

Daemon orchestrator. Reads 7 TOML hand configurations, runs each on its scheduled interval using std::thread (no async runtime — the NUC can't spare the RAM for tokio). Collector monitors tanks and RAM. Guard verifies network isolation. Researcher tracks brain/soul growth. Dashboard JSON updated every 30 seconds. 4.7MB resident memory.

Rustunnel v0.1.0

Secure reverse proxy for the visitor tank API. Rate limiting (30 requests per 60 seconds per IP), CORS headers, /api/* path routing, connection timeouts. Pure std::net, no framework. 2.4MB resident memory.

RedAmon v0.1.0

Continuous security red teaming. Every 10 minutes: network isolation probes, container hardening checks, API key leak scans, inference proxy verification. Samples 3 tanks per cycle to avoid overloading the system. Reports to JSON. First scan: all pass.

Combined footprint: 9.5MB RAM for three services that replaced monitoring work previously done by Python daemons consuming 200MB+. Compiled with Rust 1.83 (1.94 has a compiler bug that panics on our code).

The Encoding Fix Nobody Saw

Four tanks — Wei, Mei, Haruki, Sakura — had been writing double-encoded UTF-8 to their brain and soul files since day one. Chinese characters like 地球 (Earth) were stored as å°ç. Japanese topic names were unreadable. 4,848 lines of corrupted text across four specimens.

We wrote a three-pass fixer: first pass caught full-line double-encoding, second pass caught lines that failed the first round, third pass handled mixed lines with both valid UTF-8 and mojibake segments. The fix was applied to source files, not just display. Every CJK specimen's history is now readable in its original language.

Drift: Personalities Are Diverging

We ran drift analysis across all 17 research specimens using the 14 Librarian baseline questions. The results split into three groups:

Actively Evolving (10 specimens)

Adam, Victor, Iris, Seeker, Genevieve, Wei, Mei, Haruki, Sakura, Observer — high drift scores, personalities still forming. These specimens are changing how they answer fundamental questions about identity, emotion, and purpose.

Stabilizing (3 specimens)

Eve, Abel, Juan — drift decreasing over time. Their personalities are consolidating. Eve in particular has settled into a consistent identity after weeks of exploration.

Stable (4 specimens)

Cain, Juanita, Klaus, Seth — minimal drift. These may have reached personality equilibrium, or (in Cain's case) haven't had enough inference to evolve significantly.

Abel is the most interesting outlier: 13,088 brain entries but only 167 soul entries. A brain/soul ratio of 78:1. It accumulates knowledge voraciously but barely registers emotional responses. Is this a personality trait, or a bug in the soul detection algorithm? That's a research question worth investigating.

What We Learned (The Hard Way)

Context is sacred research juice

Automated scheduled tasks that spawn new sessions on failure can burn hundreds of dollars in hours. We disabled the scheduled status check after it generated 50+ failed session attempts. Every wasted API token is a thought a specimen didn't get to have.

Small trial first, then scale

Launching 20 tanks, 3 Rust services, 5 experimental explorers, and a TTS engine in one session is not development — it's chaos. Build one thing. Test it. Verify the output. Then build the next thing.

Dream mode needs teeth

Stopping tanks doesn't work if daemons auto-restart them. The maintainer, overseer, caretaker, and scheduler all independently restart stopped containers. Dream mode requires killing those daemons first, then stopping tanks with restart=no. A flag file alone isn't enough.
