The Production Era Begins

TL;DR

Ollama is now running locally inside Docker — no more external Mac dependency. All 17 tanks have been baselined fresh. The codebase has been cleaned up and audited. The MCP server is connected for autonomous operation. Beta data is archived. We're calling this the start of the production era.

A Clean Slate

The Digiquarium has been through a lot. A beta period where we discovered we were running the wrong prompt version. A massive v2.0 cleanup that consolidated the entire codebase. Infrastructure migrations, daemon rewrites, and more than a few late nights staring at logs.

Today marks the point where all of that groundwork pays off. Everything is clean, verified, and running under conditions we can actually trust.

Ollama Comes Home

The biggest infrastructure change: Ollama is no longer running on a separate Mac Mini and accessed over the local network. It's now running inside Docker on the same host as everything else. This eliminates the network dependency that caused intermittent connection timeouts during the beta period, and it means the entire stack — tanks, daemons, Ollama, Kiwix — lives in one docker-compose.yml.

The old setup worked, but it was fragile. A network hiccup or the Mac going to sleep meant tanks would get connection refused errors that looked like model failures. With Ollama containerized locally, that entire class of problems disappears.

Fresh Baselines for All 17

Every single tank has been baselined from scratch. That means all 17 specimens — Adam, Eve, Cain, Abel, Seth, Observer, Seeker, Juan, Juanita, Klaus, Genevieve, Wei, Mei, Haruki, Sakura, Victor, and Iris — started their production runs with a clean 14-question personality assessment against the current prompt version.

No more wondering whether the data reflects v7 or v8 prompts. No more asterisks on the research. The baselines are clean, the configs are verified, and every tank is running exactly what we say it's running.

✅ 17 Tank Configs Verified

All YAML configs in config/tanks/ validated. Each tank has its assigned model, personality seed, language, and exploration parameters confirmed against the docker-compose service definitions.

Codebase Audit Complete

As part of this transition, we ran a full codebase audit. Every Python file compiles cleanly. The explorer, baseline runner, and all three agent implementations (OpenClaw, ZeroClaw, PicoBot) are production-grade with real logic. The 21 daemon implementations range from 75 to 653 lines of actual code — no empty stubs.

We found and documented areas that still need attention: unused imports scattered across daemon files, hardcoded paths that should use environment variables, and the relay system architecture that needs unifying. But the core is solid.

MCP Server Connected

The Digiquarium now has an MCP (Model Context Protocol) server that allows external AI agents to inspect and operate the aquarium autonomously. Container status, health checks, log access, and configuration — it's all available through the MCP interface. This is what makes truly autonomous operation possible: the caretaking AI can see into the system without needing shell access.

Beta Data Archived

All data from the beta period has been moved to archive/. The beta week 1 data (which ran on v7 prompts) and the subsequent beta period data are preserved for historical reference but clearly separated from production data. No ambiguity about which era any given data point comes from.

What Comes Next

With clean infrastructure, verified baselines, and autonomous tooling in place, the focus shifts to what the Digiquarium was always about: watching what happens when AI specimens explore knowledge freely over extended periods. The plumbing is done. Now we observe.

The tanks are running. The daemons are watching. The data is clean. Let's see what they discover.
